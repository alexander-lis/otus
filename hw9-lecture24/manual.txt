*** Описание работы ***
Домашняя работа №9.
В качестве проектной работы была выбрана тема "Проектирование сервиса аренды электросамокатов с использованием микросервисной архитектуры"

В данном задании были спроектированы микросервисы системы.

Ниже прикреплен PDF файл со схемой, где для кажого микросервиса описано:
- Список конечных точек REST API.
- Список пользовательских сценариев.
- Список публикуемых событий и отправляемых команд по протоколу AMQP.
- Список прослушиваемых событий и получаемых команд по протоколу AMQP.
- Список таблиц базы данных.
- Сквозная нумерация пользовательских сценариев, конечных точек REST и сообщений AMQP для удобства сопоставления друг с другом.

На схеме отображены следующие микросервисы и компоненты системы:
- Auth service - сервис аутентификации и управления аккаунтами.
- Orders service - сервис бронирования самокатов.
- Park service - сервис управления парком самокатов.
- Notifications service - сервис отправки уведомлений.
- Billing service - сервис выставления счетов.
- Client application - клиентское приложение.

Взаимодействие сервисов:
- Взаимодействие клиентского приложения и сервисов осуществляется по REST.
- Взаимодействие между сервисами осуществляется по протоколу AMQP. 
- Логически сообщения AMQP разделены на события и команды. Список принимаемых команд и список публикуемых событий определяется самим сервисом.
- В качестве улучшения можно подумать в сторону написания отдельного сервиса, который будет выступать в качестве бэкенда для фронтенда с целью уменьшения количества REST вызовов от клиентского приложения.
